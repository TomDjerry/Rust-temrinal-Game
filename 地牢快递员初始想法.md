TRD：终端回合制 Roguelike《地牢快递员：契约投递》
1. 项目目标

在终端中实现一款回合制 roguelike，包含：

随机地牢生成（房间+走廊）

迷雾（FOV 视野）

怪物 AI（巡逻/追踪/逃跑）

物品系统（道具、装备、消耗品）

战斗与状态效果（中毒、流血、眩晕等）

任务系统（主线投递 + 支线合约）

存档/读档

配置与数据驱动（怪物/物品用外部文件定义）

日志与可复现随机种子（便于调试）

目标是把它做成一个可扩展的小游戏框架：后续你可以继续往上加职业、技能树、更多地图主题等。

2. 技术栈与依赖（建议）
2.1 终端 UI / 输入

ratatui：TUI 布局、面板、组件（地图/日志/状态栏/背包）

crossterm：键盘事件、raw mode、屏幕控制（ratatui 也会用到）

2.2 数据驱动与配置

serde + serde_json 或 toml：加载物品/怪物/技能/配置

config：统一处理 config 文件 + 环境变量覆盖（可选）

2.3 随机与算法

rand：RNG

可选：pathfinding（若你不想手写 BFS/A*，但建议你手写 BFS 练手）

2.4 结构与工具

thiserror：错误类型

anyhow：上层错误聚合（可选）

log + env_logger：日志（debug/trace）

ron：更适合游戏数据的格式（可选）

bincode：存档二进制序列化（或 serde_json 简单存档）

3. 游戏概念与玩法
3.1 核心循环（回合制）

每回合：
玩家行动 → 系统结算（状态/陷阱）→ NPC/怪物行动 → 视野更新 → 渲染

玩家行动包含：

移动（8方向可选，先做 4 方向）

近战攻击/远程攻击

使用物品/技能

拾取/丢弃/装备

交互（开门、开箱、读告示牌、接合约）

3.2 胜负条件

主线：找到包裹 P 并送达出口 E

失败：HP=0 或回合数超过合约期限（可选“硬时间限制”）

3.3 丰富玩法（在终端也好玩）

合约系统（Contracts）：每层随机 1-3 个支线
例：

“清理 3 只史莱姆”

“带着包裹不被击中到达出口（无伤投递）”

“在 60 回合内找到钥匙并开门”

噪音系统（Noise）：攻击/开门产生噪音，吸引怪物

光照/迷雾（FOV）：只显示可见格，探索过的格子变暗

环境交互：门、锁、箱子、陷阱、火把、毒沼

简单装备体系：武器（伤害）、护甲（减伤）、饰品（抗性/视野）

状态效果：中毒/燃烧/眩晕/流血（具有回合数与结算）

4. 功能需求（可拆任务）
4.1 地图系统
4.1.1 地图生成（必须）

生成固定尺寸网格（如 80x40 或根据终端自适应）

房间+走廊算法：

随机生成矩形房间（最小/最大尺寸可配置）

房间之间用走廊连接（L 型走廊即可）

放置：玩家、包裹、出口、怪物、道具、门、陷阱

可复现：支持指定 seed（命令行 --seed 123）

验收标准

每次开始新游戏都能生成可通关地图（玩家->包裹->出口连通）

指定同一个 seed，生成结果一致

4.1.2 Tile 类型（必须）

Wall #, Floor ., Door +, LockedDoor *, Trap ^, Water ~, Exit E

支持“是否阻挡移动 / 是否阻挡视线 / 进入触发效果”

4.2 实体与组件系统（建议用轻量 ECS 思路）

你可以不引入完整 ECS 框架，但要按 ECS 思路分清数据。

4.2.1 Entity 类型

Player

Monster

Item（地图上的物品实体）

Feature（可交互物：箱子、祭坛等）

4.2.2 关键组件（建议）

Position {x,y}

Stats {hp, max_hp, atk, def, speed}

AI { state, target }

Inventory { items, capacity }

Equipment { weapon, armor, accessory }

StatusEffects(Vec<Effect>)

Renderable { glyph, fg, bg, layer }

4.3 战斗系统
4.3.1 近战（必须）

玩家向怪物方向移动，如果该格是怪物 → 触发攻击而非移动

伤害计算（可配置）：
damage = max(1, atk - def + rng(-1..=1))

死亡处理：怪物死亡掉落物品；玩家死亡 game over

4.3.2 远程（进阶但推荐）

弓箭/法术：直线投射，遇墙停止

命中率（可选）：基于距离与敏捷

弹药/法力（可选）

验收标准

近战可用且日志可追踪每次伤害

远程攻击不会穿墙

4.4 AI 系统（重点练手）
4.4.1 AI 状态机（必须）

怪物具备状态：

Idle：巡逻/随机走

Alert：听到噪音后朝噪音源移动

Chase：看见玩家后追踪

Flee：低血量逃跑（可选）

4.4.2 寻路（必须）

至少实现 BFS（网格寻路）

每回合怪物计算下一步（可做缓存优化：只在目标变化时重新 BFS）

验收标准

玩家进入视野后，怪物能可靠追到玩家附近（不会卡墙）

噪音能吸引怪物改变路径（如果实现噪音）

4.5 视野与迷雾（FOV）

使用 Shadowcasting 或 Bresenham + 可见性（任选）

可见：亮显示；探索过不可见：暗显示；未探索：空白

验收标准

墙能遮挡视线

地图探索痕迹会保留

4.6 物品、装备、背包
4.6.1 物品类型（必须）

Consumable：药水、解毒剂、炸弹

Equipment：武器/护甲/饰品

QuestItem：包裹、钥匙

4.6.2 数据驱动（必须）

物品与怪物定义来自外部文件（JSON/TOML/RON），例字段：

id, name, glyph

rarity

stats_modifiers

use_effect（枚举）

stackable, max_stack

验收标准

添加一个新物品只需改数据文件，不改代码（至少对名称/属性/效果成立）

4.7 任务/合约系统

主线任务固定：取包裹送出口

支线合约随机抽取：kill、collect、time_limit、stealth等

合约奖励：金币/道具/属性点

验收标准

UI 中能看到当前任务进度

完成任务会结算奖励并写入日志

4.8 UI 需求（ratatui 布局）
4.8.1 面板布局（建议）

左：地图（主区域）

右上：角色状态（HP/Atk/Def/效果）

右中：任务列表

右下：事件日志（可滚动）

弹窗：背包、装备、帮助、暂停菜单

4.8.2 输入映射（建议）

移动：WASD / 方向键

g 拾取

i 背包

e 装备

u 使用物品

. 等待一回合

? 帮助

Esc 菜单

F2 快速存档，F3 读档（可选）

验收标准

无闪烁/基本流畅重绘

日志可用，至少显示最近 50 条

4.9 存档/读档

存档内容：seed、地图、实体列表、背包、任务进度、回合数

存档格式：

简单版：serde_json

更专业：bincode + 版本号

保存位置：./saves/save1.dat（或用户目录）

验收标准

存档后退出再读档，状态一致（地图与怪物位置一致）

5. 非功能性需求

性能：80x40 地图，怪物 <= 30，回合响应 < 50ms（普通机器）

可调试：

RUST_LOG=debug 输出关键系统日志

支持 --seed 复现

可测试：至少对以下模块写单测：

地图连通性检查

BFS 寻路正确性

伤害计算边界（最小伤害、减伤）

7. 里程碑与交付（强可执行）
M1：最小可玩（1-2 天）

地图生成 + 玩家移动 + 墙碰撞 + 地图渲染 + 日志

验收：能走、能看到地图、不卡死

M2：通关闭环（1-2 天）

包裹 + 出口 + 胜利条件

简单怪物（随机移动）+ HP + 近战

验收：能赢能输，有最基本挑战

M3：可玩性提升（2-4 天）

FOV 迷雾

AI 状态机 + BFS 追踪

道具/背包/装备（至少 5 种物品）

验收：怪会追你、视野遮挡成立、道具可用

M4：系统化与数据驱动（2-4 天）

怪物/物品外部数据驱动

合约系统 + 奖励

存档/读档

验收：不改代码添加新怪/新物品；存档可恢复

M5：打磨与扩展（可选）

噪音系统、门锁钥匙、更多地图主题

远程攻击/技能系统

平衡与更多 UI 功能（筛选、快捷键）

8. 验收清单（最终版）

 cargo run 可直接进入游戏（有菜单更好）

 --seed 可复现地图与内容

 主线可完成（取包裹→出口）

 至少 3 种怪物，具备追踪 AI

 至少 10 种物品（含 3 消耗品、3 装备、钥匙/包裹等任务物）

 FOV 工作正常，探索痕迹保留

 UI 至少包含：地图、状态、任务、日志、背包弹窗

 存档/读档可用

9. 风险与规避

FOV 与 AI 复杂度高：先做“简化可见性（射线检测）+ BFS”，再替换为 shadowcasting

TUI 输入与刷新坑多：ratatui+crossterm 走标准范式（raw mode + tick loop）

系统耦合：坚持“系统”拆分，world 只存数据，系统只做更新
