# 环境交互实现计划

> Goal: 在不重写主循环的前提下，加入门、固定伤害陷阱与基础噪音交互，并保持现有 AI、FOV、支线 stealth 联动可回归验证。

## Task 1: 定义环境数据模型
- 修改地图与核心类型，加入门状态与陷阱数据
- 保证旧存档缺失字段时可安全恢复默认值
- 添加数据层测试或构造测试

## Task 2: 接入门的移动与视线规则
- 关闭门阻挡移动
- 关闭门阻挡 FOV，开启门允许通过与透视
- 为门状态相关判定补测试

## Task 3: 实现开门与关门动作
- 撞上关闭门时改为开门交互
- 新增显式关门动作与按键映射
- 成功开/关门时消耗回合并制造噪音
- 无可关闭门时只写日志不消耗回合

## Task 4: 实现陷阱触发逻辑
- 玩家进入陷阱格后立即触发
- 造成固定伤害并制造较大噪音
- 触发后失效，避免重复伤害
- 补触发一次性与伤害测试

## Task 5: 接入 AI / stealth 联动
- 门和陷阱噪音走现有噪音链路
- 验证怪物进入 Alert 的行为不回归
- 验证 stealth 合约会因环境噪音导致的 Alert 而失败

## Task 6: 补快照与 UI 显示
- 快照输出开门、关门、陷阱状态
- UI 地图正确显示环境元素
- 补快照/UI 断言测试

## Task 7: 全量验证与文档同步
- 运行 `cargo fmt`
- 运行 `cargo test`
- 运行 `cargo clippy --all-targets --all-features -- -D warnings`
- 运行 `cargo build`
- 更新 `docs/project-progress.md` 与必要的 README 说明
- 提交并推送远端
